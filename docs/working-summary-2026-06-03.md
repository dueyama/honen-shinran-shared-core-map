# Okyou2 Working Summary 2026-06-03

## 位置づけ

`Okyou2` は、完了済みの先行 `Okyou` の後継プロジェクトだが、継続開発ではない。先行 `Okyou` はリリース済み成果物として閉じ、参考として読むだけにする。先行 repo は変更しない。

今回の方向は、先行 `Okyou` の意味埋め込み・ビューア・論文的まとめを踏まえつつ、浄土教、とくに法然・親鸞・浄土系典籍の関係を、より研究上説明できる形で見直すことにある。

## 当初の発展案

有力な発展方向として、次が挙がった。

- コーパス拡張: 宗派別に祖師文献・注釈・勤行文を増やす。
- 方法検証: chunk size、自然単位チャンク、別 embedding model、PCA 以外の可視化を試す。
- 三層分析強化: 意味、文体語彙、典拠マーカーを分離する。
- 親鸞/浄土系への集中: 『教行信証』巻別、浄土三部経、異訳、引用研究と照合する。
- 公開 viewer 強化: filter、近傍表、巻別表示、層別表示を入れる。

このうち、すぐに動かす筋としては「意味マップに法然・親鸞・高僧文献を重ね、あとで三層分析へ展開する」が良い。

## 阿弥陀経・異訳・写本について

阿弥陀経を網羅的に見る案も検討した。漢訳としては、鳩摩羅什訳『仏説阿弥陀経』と玄奘訳『称讃浄土仏摂受経』が基本線になる。後者は鳩摩羅什訳『阿弥陀経』の異訳本として説明される。

ただし、「漢訳が二つ」という話と、「写本・書写による異本が多数ある」という話は別である。研究上は次のように分ける必要がある。

- canonical translation: 大正蔵/SAT などで扱える漢訳テキスト。
- manuscript witness: 古写経、断簡、敦煌写本などの個別資料。
- variant reading: 写本間・版本間・校訂本間の異同。
- usable text: embedding や差分照合へ投入できる機械可読テキスト。

写本画像が見られることと、本文テキストとして取得できることは別問題である。画像だけなら参照できても、OCR/翻刻/校訂がなければ、今回の埋め込み分析にそのまま入れるのは難しい。

## チベット大蔵経・敦煌系資料

東北大学のデルゲ版経典DBは、デルゲ版チベット大蔵経の画像イメージを提供するデータベースとして確認した。ページ上の説明では、基本的に画像提供であり、本文テキスト検索はない。IIIF 準拠の高品質画像として使えるが、現段階では embedding 用の本文テキスト取得元とはみなしにくい。

敦煌系資料は「チベット」そのものではない。敦煌文書は地域・出土資料群であり、中国語、チベット語、サンスクリット、コータン語、ソグド語など複数言語を含む。したがって、敦煌系阿弥陀経を扱うなら、まず資料番号・言語・画像/翻刻の有無を確定する必要がある。

結論として、阿弥陀経の写本・敦煌・チベット系を網羅的に扱うのは魅力的だが、最初の実験対象としては重い。まずは法然・親鸞・浄土系祖師文献の意味マップを作り、その後で阿弥陀経異訳・写本系へ進む方がよい。

## 藤原智論文との関係

藤原智「『教行信証』における引用文について：古写経本による再検討」は、『教行信証』の引用文を、版本大蔵経だけでなく日本古写経の観点から再検討する研究として位置づく。書誌上は、2020年、『近現代『教行信証』研究検証プロジェクト研究紀要』3号、63-107頁。

今回の手法で再検討するなら、直接の再現ではなく、次のような補助的再検討が現実的である。

- semantic layer: 『教行信証』内の引用群が、どの経論・注釈群に意味的に近いかを見る。
- source-marker layer: 経名、人名、固有句、引用導入句、典拠句を抽出し、藤原論文の指摘と照合する。
- style layer: SAT/版本系テキスト、古写経系翻刻、和文訓読テキストが混じったときの文体差を検出する。

限界は明確である。古写経本文そのものが機械可読で広く取得できるとは限らない。画像資料や影印本を得ても、それを公開・再配布してはいけない。したがって、公開可能なのは、本文を含まない座標、距離、対応表、要約統計、可視化、方法記述に限定する。

## 法然と親鸞を比較する筋

法然の代表書としては『選択本願念仏集』、親鸞の代表書としては『教行信証』を置くのが自然である。両者は浄土教の中心問題を共有し、法然は親鸞の師であるため、意味的には大きく重なるはずである。

一方で、比較として面白いのは「重なる」ことではなく、重なった上でどこにズレが出るかである。『選択集』も経論釈の引用を軸にするが、法然は選択本願・専修念仏の論証に収束しやすい。『教行信証』は巻別構造、経論釈の大量引用、信・証・真仏土・化身土などの展開を持つため、より広がる可能性がある。

このズレを読むために、まず 2D の意味マップを作り、平均だけでなく広がりも見ることにした。

## 実験1: 法然・親鸞の意味マップ

最初に character n-gram TF-IDF + SVD の粗い地図を作ったが、これは埋め込みではないため、主分析からは外す。以後は先行 `Okyou` と同じ方針に寄せ、OpenAI embedding API を使う。

先行 `Okyou` の方法を確認したところ、基準は `text-embedding-3-large`、`tiktoken cl100k_base`、700 token chunks、100 token overlap だった。そこで `Okyou2` でもこの方法を踏襲した。

対象:

- 法然『選択本願念仏集』: 聖教電子化研究会 seiten HTML から取得。
- 親鸞『教行信証』: 聖教電子化研究会 seiten HTML から取得。

処理:

- `honbun`, `s_sage`, `l_nobegaki` の日本語読み本文を使用。
- `r_kanbun` は重複を避けるため初回では除外。
- 700 token chunks、100 overlap。
- `text-embedding-3-large` で埋め込み。
- PCA は NumPy SVD で実装。

結果:

- 『選択本願念仏集』: 103 chunks。
- 『教行信証』: 301 chunks。
- PCA explained variance ratio: PC1 `0.063873`, PC2 `0.046549`。
- 2D 上の重心距離: `0.173264`。

地図では、法然と親鸞は大きく重なりつつ、親鸞が右方向に広く伸びた。右方向の親鸞 chunk を見ると、『大般涅槃経』、阿闍世、五逆、提婆達多、慚愧、外教批判、老子・孔子・史記・論語などが出ていた。つまり、共有核は善導・観経・正行/雑行・専念・深心・往生あたりで、親鸞の広がりは罪・救済・涅槃経・外教批判・化身土方面に出ている可能性がある。

ただし、PCA の軸の符号は任意であり、PC1+PC2 は全分散の約 11% しか説明しない。ここでいう「右」は、図上の方向であって、絶対的な意味軸ではない。

基準:

- PCA fit は法然『選択集』と親鸞『教行信証』の chunks のみに限定。
- 高僧文献は同じ embedding 空間から、その固定 PCA 面へ投影。

## 実験3: 親鸞だけが入り込むゾーン

高僧アンカー図を見た上で、親鸞点だけが入り込んでいるように見える領域を初回抽出した。判定は、親鸞 chunk から見た最寄り非親鸞点、つまり法然または高僧アンカー点までの 2D 距離が `0.10` 以上のものとした。この閾値では親鸞 301 chunks のうち 20 chunks が拾われる。

初回分類では、孤立領域は一枚岩ではなく、少なくとも次の五つに分かれた。

- 行巻・真仏土の上方島: 名号、本願、無量寿、阿弥陀、不退。
- 信巻右下の阿闍世・涅槃経島: 五逆、提婆、耆婆、慙愧、地獄。
- 化身土末の護法・鬼神・宇宙秩序島: 大集、鬼神、星宿。
- 化身土末の外教批判島: 老子、孔子、史記、周書、論語、道士。
- 化身土本の方便・雑行島: 方便、雑行、専修、回向。

この結果から、親鸞の広がりは単純に「法然から離れる」ではなく、共有された専修念仏の中心から、名号/本願力、罪と救済、末法/護法、外教批判、方便真門の整理へ分岐していると見る方がよい。

詳細は `docs/shinran-isolated-zones-2026-06-03.md` に記録した。本文は出力していない。

## 現時点の研究の筋

現時点で良さそうな筋は、次である。

1. 法然・親鸞の意味マップを固定基準にする。
2. 七高僧・浄土三部経・主要注釈を同じ面に投影する。
3. 親鸞の広がり方向にある chunk を、巻別・引用別に分解する。
4. その領域に近い高僧文献・経論を抽出する。
5. source-marker layer で、実際の引用・経名・人名・固有句を照合する。
6. style layer で、テキスト種別や漢文/和文読みの混入効果を確認する。
7. viewer では、作品別、巻別、層別、近傍表、chunk 内容の安全な要約を表示する。

この筋なら、「親鸞は法然と意味的に重なる」という当然の前提を踏まえつつ、「どの方向に広がるか」「その方向にどの祖師・注釈・典拠が近いか」を説明できる。

## 次にやるとよいこと

優先順位は次の通り。

1. 親鸞『教行信証』を巻別に色分けする。
2. 右方向・右下方向・上方向など、地図上の特徴領域ごとに代表 chunk を抽出する。
3. 高僧アンカーの近傍表を作る。
4. 浄土三部経を追加し、法然/親鸞/高僧/経典の四者関係を見る。
5. source-marker layer の最小版として、経名・人名・引用導入句の辞書を作る。
6. 公開 viewer では raw text を出さず、座標、出典ラベル、本文なしの要約統計、必要なら短い安全要約だけを表示する。

## 後半更新: 漢文テキストと chunk 方針

途中で、解析本文は日本語読みではなく漢文を基本にする方針へ切り替えた。現在の accepted baseline は次である。

- 法然『選擇本願念佛集』: SAT `T2608`。
- 親鸞『顯淨土眞實教行證文類』: SAT `T2646`。
- embedding model: OpenAI `text-embedding-3-large`。
- embedding dimension: `3072`。
- tokenizer: `tiktoken` `cl100k_base`。
- chunk: 700 token、100 token overlap。
- chunk 作成: `tiktoken.decode_with_offsets()` を使い、Unicode 文字境界で切る。

SAT 漢文の準備では、T2608 は 1644 SAT lines、27246 compact chars、T2646 は 4215 SAT lines、76030 compact chars だった。品質チェックでは `U+FFFD`、HTML tag/entity、画像プレースホルダ、ボタン文字列、SAT line ref の本文混入はいずれも 0 とした。

前稿と同型の固定 token slice をそのまま `encoder.decode()` する方法は、chunk 端で `U+FFFD` を作る場合がある。これは 64-bit の問題ではなく、token boundary と Unicode 文字境界のズレである。比較実験では、古い方法は 260 chunks 中 148 chunks に `U+FFFD` が入ったが、新しい Unicode-safe 方法では 0 だった。一方で old/new centroid cosine は法然 `0.999352`、親鸞 `0.999443` で、大局的な地図は大きくは動かなかった。

したがって、前稿や旧試行の図は広域の見取り図としては参考にできる。ただし、今後の出版・報告用の基準処理としては、本文品質、再現性、line range との対応を優先し、Unicode-safe chunk を使う。

## 後半更新: SAT safe 高僧アンカー図

SAT 漢文・Unicode-safe chunk で、法然/親鸞を focus texts とし、龍樹・天親・曇鸞・道綽・善導・源信をアンカーとして重ねた。

focus chunks:

- 法然: 69 chunks。
- 親鸞: 191 chunks。

anchor chunks:

- 龍樹: 6 chunks。
- 天親: 9 chunks。
- 曇鸞: 31 chunks。
- 道綽: 42 chunks。
- 善導: 23 chunks。
- 源信: 73 chunks。

PCA は法然・親鸞の focus chunks だけで fit し、高僧アンカーは後から同じ面へ投影した。元の embedding は 3072 次元で、PC1+PC2 は約 11.6% の分散を説明する。2D 上の親鸞重心への距離は、道綽、善導、源信、曇鸞、天親、龍樹の順で近かった。

2D 投影法は PCA だけではない。UMAP、t-SNE、MDS などを使うと局所近傍やクラスタの見え方は変わり得る。今回は、線形で安定しており、高僧アンカーを同じ fit 面へ投影しやすい PCA を基準表示とした。別投影で同じ方向性が出るかは今後の方法検証に回す。

法然/親鸞の二群が最も重なるような投影にも意味はある。ただし、それは共通核を強調する目的付き図であり、中立な地図ではない。無制約に重なりを最大化すると差異方向を捨てるだけになりうるので、主図とは分けて「共通核強調図」として扱うのがよい。逆に差異強調図も別途作れる。

高次元 cosine の近傍集計では、親鸞 chunks の最近傍アンカーは、源信 58、道綽 58、曇鸞 27、善導 24、天親 22、龍樹 2 だった。法然も比較対象に含めると、親鸞の最近傍非自己は法然 48、道綽 45、源信 36、曇鸞 23、天親 21、善導 17、龍樹 1 となる。

法然 chunks から見ると、最近傍非自己は親鸞 50、道綽 11、源信 4、善導 3、曇鸞 1 だった。高僧アンカーだけに限定すると、道綽 28、源信 24、善導 11、曇鸞 5、天親 1 である。

この結果は、「法然と親鸞は当然かなり重なる」という前提を支持する。歴史的には、親鸞が法然を見ていると解釈するのが自然である。ただし、embedding だけで影響関係を証明するわけではないので、論文的には「歴史的関係を前提にした意味空間上の重なり」と書くのが安全である。

## 後半更新: 法然のはみ出しと書物形式

法然側にも、親鸞・高僧アンカーから少し離れて見える領域がある。これを調べたところ、単純に「念仏の意味が親鸞と違う」というより、`選択`、`正雑二行`、`諸行との取捨`、`一向専念`、`三心/真実心`、`善導/観経`、`付属/証誠` などがまとまって出ていた。

現時点の読みは次の通り。

- 法然『選択集』は、諸行の中からなぜ念仏を選ぶのかを論証する本として見える。
- 親鸞『教行信証』は、法然の専修念仏を受けた上で、それを経典・論釈・高僧の歴史的/典拠的流れに置き直し、教・行・信・証・真仏土・化身土へ展開する本として見える。
- したがって、法然のはみ出しは「念仏そのものの別解釈」よりも、「念仏を唯一的・専修的に選び出す分類・廃立の論証」の圧縮として読むのがよい。

短く言えば、法然は「念仏を選ぶ論理」を構成し、親鸞はその念仏を「本願・信・証の歴史的/典拠的体系」として展開する、という対比が今の map からは自然である。

注意点として、法然の最も孤立して見える一部 chunk には末尾・奥書系の情報が混じる可能性がある。思想的な「はみ出し」として読む場合は、この種の chunk を除外して再集計する必要がある。

## 後半更新: 法然側の corpus 方針

法然に『教行信証』と同じ性格の一冊を探すと、厳密には見つかりにくい。最も近い主著はやはり『選択本願念仏集』だが、『教行信証』のような巨大な巻別体系書というより、称名念仏を選択する論証の書である。

法然側を広くするなら、次のように扱うのがよい。

- 主軸: 『選択本願念仏集』。
- 法然 corpus: 『黒谷上人語灯録』系の法語・消息・漢語/和語資料。
- 講説・経釈系: 『三部経釈』、とくに『無量寿経釈』『観無量寿経釈』『阿弥陀経釈』。

ただし『黒谷上人語灯録』は法然本人が一冊の体系書として書いたものではなく、道光による蒐集編纂物である。『三部経釈』も現存テキストの伝来に注意が必要である。したがって分析単位としては、`親鸞『教行信証』` 対 `法然『選択集』 + 語灯録系 + 三部経釈系` と置くのが現実的である。
